### Support Macro: Campaign Rejection
**Purpose:** Help advertisers remediate rejected campaigns.  
**Scope:** Policy-based rejections.  
**Template Highlights:**  
- Include exact rejection code and a plain-language reason.  
- Provide one compliant example revision.  
- Offer expedited re-review once updated assets are submitted.  
**SLA:** Re-review target **2 business hours** for corrected submissions.  
**Owner:** Policy Support  
**Last Updated:** 2026-01-31

**Tone and Communication Rules:**
- Use clear, non-accusatory language.
- Confirm issue scope before proposing next steps.
- Avoid exposing internal risk-scoring or anti-fraud internals.

**Required Data Before Escalation:**
- Customer account identifier
- Reproducible timestamps and affected object IDs
- Relevant screenshots or exported report snippets

**Escalation Matrix:**
- Billing -> Finance Operations
- Delivery quality -> Ad Delivery Team
- Access/security concerns -> Identity/Security

**Quality Bar:**
- First response must include expected SLA and next update time.
- Resolution summary must include root cause class and prevention note.
